# Contributing

Pull requests are welcome. The repo's bias is toward content that helps a *working PM* do the next thing on their desk, not classroom or case-interview prep.

## What's in scope

- Frameworks for product decisions, prioritization, validation, metrics, launch, and discovery
- Templates for artifacts that go up the chain (PRDs, strategy memos, decision logs, post-mortems)
- Rubrics for grading PM artifacts
- Claude Code skills that operationalize the frameworks
- Cross-functional patterns (engineering, design, sales, support partnerships)

## What's out of scope

- Case-interview prep
- Personal productivity tips disconnected from PM artifacts
- Tool recommendations or competitor positioning
- Anything that requires personal context the reader can't replicate
- Anecdotes from specific class sessions, named individuals, or unpublished sources

## Style

- Plain verbs. Direct prose. No em-dashes (sparingly OK in parens).
- 3-5 options before single recommendations.
- Behavioral signals over attitudinal ones.
- Falsifiable success criteria when possible.
- "Honest 'I don't know' beats fabrication."
- Match the voice of [`what-makes-a-good-pm.md`](what-makes-a-good-pm.md) for posture docs and [`SKILL.md.tmpl`](SKILL.md.tmpl) for new skills.

## How to submit

- For a new framework, doc, or section: prefer **appending to an existing file** when the content fits naturally. Create a new file only when the topic is genuinely separable.
- Open a PR with the addition or change. Reference the file structure and explain the overlap check against existing content.
- If you're not sure whether it fits, open an [issue](https://github.com/kalyvask/pm-evaluation-framework/issues/new/choose) first with the *Framework or doc suggestion* template.

## How to give feedback on a skill

If you've invoked one of the Claude Code skills on a real PM artifact and have specific observations on what it did well or missed, open an issue with the *Skill feedback* template. Concrete examples (sanitized) are the highest-signal contributions to the skill library.
